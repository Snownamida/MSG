## 2026-07 初 — PoC：单句 patch 中文

把句 62 patch 成中文 8×8 字模，Mesen 逐帧核对：
- **确认**：tile 恒等映射（屏上 tile `$B7` = CHR tile `$B7`）；每场景各自加载字库 bank
  （标题=bank0、海边=bank11、机甲=bank48）；字模位平面格式（plane0 全 FF，笔画=清 plane1 的位）。
- **三个坑**：① 8×8 太糊，笔画多的字糊成一团 → 需 16×16；② 字典字符串重叠（极致 MTE），
  就地覆写踩坏邻居 → **回写必须整体重建到空闲区，绝不就地覆写**；③ CG 与字库是不同 CHR bank，
  最初"机甲变中文"是写错了 bank → **只往对话框 bank0 写，机甲完好**。
  ⚠ 工具坑：Mesen **Tilemap Viewer 的 CHR 地址不追 MMC5 bank**，判 bank 要用 **Tile Viewer**。

### 里程碑 2 — 单句正确回写（实机）

`reinsert.py`：重建到空闲区 + text 指针反解 + 块串就地重建 + 字模只写 bank0。句 62 清晰中文、
机甲毫发无损、零副作用。单场景 demo 无需扩 CHR。但两上限仍在：256 tile/bank、8×8 太糊。

### 里程碑 3 — B 命门：文本段切到扩 CHR 专属 bank（实机）

对话框是分屏 region-1（bank=`$0451`）。单处等长改 `0x7EB8F` 的 `LDA $0451`→`LDA #128`，
使对话框读扩 CHR 的 bank 128。`build_b_demo.py` 实机：对话框中文、机甲完好。
→ **全部技术难关验证通过**，剩余为工程量。

### 16×16 改造尝试（尝试 1/2，未成但定位真凶）

两次改 `$F135` 字符发射循环做 16×16，实机均无效。**Lua 铁证**：我的 patch 确实映射并执行，
但填 `$04A8` 的不是它。**真凶：对话正文由 bank 62 独立打字机 emit 渲染**（`$D2ED`），不经
`$F135`（那是名字/菜单排版）。教训：真 16×16 要改整个 bank62 子系统，非"覆写一条例程"。
另发现**幻影空槽事故**：`0x8B54+` 是句 1108 数据、非空槽，误写会潜伏损坏
→ **副作用检查必须全量 2781 句**（40 句抽查会漏）。

### 硬骨头③ — 句级切换字库 bank（实机验证）

让溢出句（187）用第二字库 bank。方案演进：WRAM 路线（游戏 0 写入但写保护未开、真机可能无 RAM）
→ **ExRAM 路线**（`$5FA5-$5FFF` 动态全章零写入、`$5104=2` 仅 reset 设一次）。
detour `$F081`（读句指针例程尾，等长 `JSR $5FA5`）+ ExRAM 例程维护 shadow + NMI 改读 shadow +
bank127 开机 shim 拷入。四个连环坑，全部实测抓获：
1. **bank 切换必须无缝交接**：shim 尾部做 `STA $5117` → 切换后下一条指令从 bank63 同地址取 =
   无关代码 → 上电即崩。修：redirect 挪早一条指令，`JMP` 回原位切换（两 bank 该处内容一致）。
2. **通用例程需强守卫**：`$F06F` 被渲染层复用（$87 指向其他表、某些 bank 首字节恰=0），
   B 句显示中途 shadow 被重置 → 乱码。修：`$88∈[$BC,$C0)` 范围守卫。
3. **表项 ±2 容差**：引擎显示期间以表项 +1/+2 偏移重读指针，精确匹配漏掉。修：lo 差 <3 判命中。
4. **存档快照包含 ExRAM**：旧 checkpoint 把旧例程灌回去，"修复无效"白查一轮。修：payload 一变
   即重新生成 checkpoint。
另捕获**全局 off-by-one**：Lua trace 的 N 标签 = msgtool 句号 −1（实证：lastN=186 时屏显 tr[187]）；
build 自洽（tr[n]↔slot n），B-list 指针直接由 `sentence_pointer(n)` 换算免疫。
副产物：修 `mesen.py run_lua` stdout 管道静默失败（改临时文件重定向）。
实机：句 187 两页爸爸戏完整中文（bank $A1），shadow 全程稳定，菜单/普通对话/开机全回归。
