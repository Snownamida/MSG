# 文本系统 —— 两级字典压缩

《Metal Slader Glory》是当年唯一的 1MB FC 卡带，文本量巨大，剧本在 ROM 里以**两级字典压缩**
（MTE，Multiple Tile Encoding）存储。本项目把全部 **2781 句**日文剧本逆向解出为可读文本，
是汉化的第一步。工具见 [`../src/msgtool.py`](../src/msgtool.py)（零依赖单文件解析器）。

地址约定：`PRG` = PRG ROM 内偏移（= .nes 文件偏移 − 0x10）；文件偏移含 16B iNES 头。

## 数据流

```
句子编号(1~2781) ──算术──▶ 句子指针 ──读3B──▶ 句子
    ──bank运算──▶ 文本块串指针 ──读到0x00──▶ 文本块串(block string)
    ──拆分──▶ 文本块 block（<0x80 单字节 / ≥0x80 双字节；即 MTE 字典项）
    ──查表──▶ 文本指针 ──读3B──▶ 文本
    ──位运算解包──▶ 字符串地址+长度 ──读──▶ PPU 编码串
    ──码表──▶ Unicode 文本
```

## 关键常量（逆向确认）

| 结构 | 偏移 | 说明 |
|---|---|---|
| 句子指针表 | `0x1CC5` | 2781 条 × 3B |
| block → 文本表 | `0x3D5C` | 双字节 block 槽止于 `idx=0xB53`（`0x5F58`），之后是句 1108 数据 |
| PPU 串区（可回写） | `0x5F60–0x76000` | `string_pointer` 可达范围约 `0x4A010–0x76010` |
| 块串区 | `0x76000–0x100010` | 扩 PRG→1MB 后 `block_string_pointer` 可达 `0x10E00F` |

## 编码层

- **PPU 编码字节 b → 字库 tile 号 = b，恒等映射**（无偏移、无查表）。逐环验证见
  [`ENGINE.md`](ENGINE.md#tile-映射恒等) 的渲染链。
- tile 排列 = `msgtool.py` 的 `CHARSET` 顺序（tile `0x20`='0'、`0x2A`='A'、`0x50`='あ'、
  `0x0E`='゛'、`0xFE`=空白）。
- **控制码**（块串层）：双字节 `05/07/0B/0C/13/14/16/18`、三字节 `0F/10/12/17`。
  它们**不在字形渲染例程分派**，由上游排版层（bank 61/62，入口 `$A4D7`）经跳转表处理，
  语义为行/页级：换行、翻页等待、光标定位、角色名、立绘、颜色等。宽度上限 `#$1E`=30 tile/行。
- **浊音/半浊音**：PPU 串层的 `0x0E`/`0x0F` 前缀，渲染时摆到假名上方那格（硬件叠放，非合成字形）。

## 结构块（携带演出语义）

PPU 串以 `0x00` 开头的块是**结构块**，渲染为空但块 ID 本身有语义，**回写时绝不能挪用为字模**：

| 类型 | 判据 | 语义 |
|---|---|---|
| 设置块 `{sHH}` | PPU 全 0 | 区域/对话框初始化 |
| 空隙块 `{gHH}` | PPU=`[00,N,FE*N]` | 渲染 N 个空格；**块 ID 决定该行属性/调色板**（绿字行属性 $FF 靠行首空隙块 `0x4B`） |
| 名字块 | `[00,X]` 前缀 | 说话人颜色/缩进/名字 |
| 折行锚 `0x02` | 空块 | 换行 + 分页（配合 30 列阈值） |

结构化导出/回写把它们保留为可见 token（`/`、`{sHH}`、`{gHH}`、`~XXXX~`），
见 [`../src/structio.py`](../src/structio.py) 与 [`LOCALIZATION.md`](LOCALIZATION.md)。

## 工具用法

```bash
python3 src/msgtool.py export      # 导出全部 2781 句日文剧本（含 ~控制码~）
python3 src/msgtool.py blocks      # 导出 MTE 字典（全部文本块）
python3 src/msgtool.py used        # 用 .cdl 统计游戏实际用到哪些句子
python3 src/msgtool.py map         # PRG 字节用途图谱（找空闲空间）
```
