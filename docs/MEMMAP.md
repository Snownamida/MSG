## 分屏渲染机制(2026-07-08 反汇编+实机确认)★核心

屏幕竖分多区(region),每区一套 CHR bank。区 bank 存 shadow 数组 **$045E-$0461**(=region0/1/2/3):
- NMI(每帧)从 $0450-$0453 拷入 $045E-$0461:`$EB79 LDA$0450;STA$045E` / `$EB7F LDA$0451;STA$045F`(汉化`ORA#$80`) / `$EB85 LDA$0452;STA$0460` / `$EB8B LDA$0453;STA$0461`。
- **NMI $EB58 `LDA $0450,X; STA $512B/$5123`** 设**顶部 strip** bank:X=0→$0450(=CG),X=2→$0452。X 由 $EBC7,Y 表 + $044E 决定。
- 分屏 IRQ($F978→`JMP($0633)`动态派发)+ **$FCC9/$FF34 `LDA $045E,Y; STA $512B/$5123`** 按区号 Y 应用各区 bank。
- 实测:普通场景 2 区(region0=CG=$045E / region1=对话框=$045F);**场景67 三段**:顶部绿字 strip(=$0452)+中间CG portrait(region0=$045E=67)+底部对话框(region1=$045F=E7)。
- **region2/3($0460/$0461)全程 =0/未渲染**(attract/海边/穿梭机实测);绿字顶部 strip 走 $EB58 X=2→$0452,**非** $0460。

| bank | 内容 |
|---|---|

| 地址 | 用途 |
|---|---|
| `$E9C3` | NMI 入口(向量 $FFFA) |

| `$EB7F`(file 0x7EB8F) | **region1 字库钩子**:原`LDA$0451;STA$045F`→汉化`LDA $5FFF;STA $045F`(读 ExRAM shadow;默认=$0450\|0x80,B句时=B bank) |
| `$EB85`(file 0x7EB95) | **region2 钩子(绿字修复)**:原`LDA$0452;STA$0460`→汉化`LDA#$E0;STA$0452;NOP`(每帧置$0452=GREEN_BANK) |
| `$F06F-$F084` | **"经($87)读3字节"核心例程**(bank63):`LDY#0; LDA($87),Y ×3 → $80/$81/$82; LDA $7E; STA $5115; RTS`。句子指针读取必经;$F071 是 exec 观测点 |
| `$F081`(file 0x7F091) | **句级切bank detour**:原`STA $5115`→汉化`JSR $5FA5`(ExRAM例程;此刻 $5115 仍=表bank,$A000 可验表) |
| `$F978` | IRQ 入口(向量 $FFFE)→分屏引擎(`JMP($0633)`动态派发,句柄表$FCA8/$FCAE) |
| `$FCC9/$FF34` | 分屏应用区 bank:`LDA $045E,Y; STA $512B/$5123` |

## 句级第二字库 bank(硬骨头③,2026-07 已实机验证)

超单场景bank容量的句(如187)放独立 B bank,运行时按当前句切换。全链:

- **ExRAM 布局**(`$5FA5-$5FFF`,实测全章含密语屏零写入;$5104=2 仅 reset 设一次→放代码安全):
  `$5FA5` 例程(0x43B) / `$5FE8` B-list(ptrHi,ptrLo,bank ×≤7 + 00终止) / `$5FFF` SHADOW(NMI每帧读)。
- **例程逻辑**:验主表($A000首字节==0)→ **守卫 $88∈[$BC,$C0)**(此例程被渲染层复用指向其他表,
  某些bank首字节恰=0,没守卫会在B句显示中途重置shadow→乱码)→ 默认 shadow=$0450|0x80 →
  扫B-list:hi精确匹配 + **lo ±2容差**(引擎显示期间会以表项+1/+2偏移重读,精确匹配漏掉)→ 命中=表项bank。
  寄存器契约:A/X 压栈;出口 PLA 复刻原 `LDA $7E` 的 N/Z。
- **开机注入**(ExRAM 是 RAM,ROM 补丁进不去):上电 `$5117=$FF`→`$E000`=bank127(扩PRG时拷的bank63
  副本,切换后永不执行=8KB自由区)。bank127 里 `$E9B1`→`JMP $E800`;`$E800` shim 把 payload($E820)
  拷入 `$5FA5`,补做被顶掉的 `LDA#$BE;STA$5114`,`JMP $E9B6` 回原切换指令。
- **B bank 内容**:bank0拷贝 + 固定/跨场景/名字字(全局码,与A bank同码位→切换瞬间共显安全)+
  该句独有字(独占整个剩余码池)。实测选话题时对话框先清屏→深话题句独占屏幕,共显集极小。
- **B-list 指针**:直接 `0xA000 + R0.sentence_pointer(n) - 0x10`。⚠ Lua trace 的 N 标签 = msgtool 句号 −1
  (表基 $BCB5 = msgtool句1 的表项;`(p-$BCB5)//3` 是 0-based)——勿用 Lua 公式反推表项地址。
- ⚠ **存档快照包含 ExRAM**:ExRAM payload 一变,旧 checkpoint 会把旧例程灌回去→改完必须重新生成
  全部 checkpoint(排查时曾因此"修复无效"白查一轮)。

## 密语/CONTINUE 机制(2026-07-08 完全确认+已修)

- **密语 = "えりな もう おきなよ"**(码 え53 り77 な64 も72 う52 お54 き56 よ75),句154(场景14 艾莉娜口述)教给玩家。
- **CONTINUE 屏**(标题选 CONTINUE,场景00):顶部忠回忆独白(**句22** "‥‥ あー よく寝た‥‥エリナ…おこしてくれって…")+ 中间蓝框密语输入 + 底部**全假名选择格**(あ-ん/ア-ン/浊音/符号/ススム前进/オワル)。全用 **bank0** 渲染。
- **存档屏**(场景14 选休息→场景00):句15/17/19/20。
