## 字库容量
- CHR bank = 4KB = 256 tile 槽;扣固定码(名字/命令/标点/假名)后 **每 bank ~200 独有汉字**。
- 第二章去重汉字 822;7 个大场景超 213:**68=359, 38=328, 5F=278, 4B=265, 5C=240, 28=220, 57=215**。

## 分配(实测可行:346句→40bank,空闲池101,富余)
- 默认:字库 bank = `场景|0x80`(复用 build_ch1_mb 的 NMI 单点改,已在 ROM)。
- 溢出:大场景按剧情顺序(句号序)贪心切多 bank,超 ~200 独有字开新 B bank。多 bank 场景:
  `68(4), 38(3), 4B(3), 28/2F/57/5C/5F/62(各2)` = 13 个溢出 B bank。
- B bank 逐句切换:复用 build_ch1_mb 硬骨头③ 的 ExRAM 钩子($5FA5,shadow $5FFF),把逐句→bank 写进 B-list。
  ★同 bank 相邻句共显安全(按句号顺序分组,不跨话题边界拆)。

## 与第一章的场景重叠(统一 012 构建)
- **00**:两章都跳过(保原版假名密语/存档屏,勿改)。
- **5C**:第一章仅瞬态 CG(检查是否装了 5C 对话);第二章机场主用(240字)。bank DC 存第二章 5C 字。
- **67**:第一章绿字(~104字,走 GREEN_BANK $E0)+第二章 67(2字);并集小,合并即可。
- 注意 `场景60|0x80=$E0` 撞 GREEN_BANK → 第二章场景60 改用空闲 bank(或 GREEN 换号)。

### Phase 2(待:啃溢出句到完美)——第二章真·硬骨头
- 需 ~147 溢出句 → ~13 B bank。**当前 ExRAM 钩子内联比较每 run ~15字节,90字节上限仅容 ~3-4 run**,远不够。
- ★ExRAM 空间修正:$5C00-$5D18 虽在对话场景零写入,但 **CG 过场场景(开场/穿梭机)很可能用 MMC5 ExRAM 扩展图形→会写它**(build 作者只信 $5FA5+ 正因此)。故表**不能**放 $5C00。
- 方案:**紧凑表驱动钩子**(表放现有安全区 $5FA5-$5FFF):
  · 表项 3字节(hi, lo_start, bank;span 固定=3 或编进 hi 高位),钩子循环比 $87/$88。
  · 循环代码 ~20B + 基础(追加表/主表默认)~25B + 尾 6B ≈ 51B → 余 ~39B 存表 = **~13 条**,够。
  · 现有 91B 安全区经 CG 场景实测存活(第一章 demo 全程 CG 后仍正确)→ 表在此安全。
- 实现要点:$5FA5 例程改为"默认 shadow=$0450|$80 后,循环查表命中则改 shadow=表项bank";B-list 由 build 生成(逐句→bank,同 bank 连续句号并 run)。

### ★Phase2 更深发现(2026-07-10,实测)——两个新拦路
1. **句表分 bank0/bank1,硬骨头③ B-list 仅支持 bank0**:`sentence_pointer(n)=3*(n-1)+0x1CC5`;n>~281 时 `_pf≥0x2010` 落 bank1 句表($5115=bank1,$A000 首字节=sig≠0)。当前钩子只认 sig=0(bank0)+ sig=$36(追加表);**第二章句多在 282-704=bank1 表**,B-list 的 `assert _pf<0x2010` 会失败。→ 钩子须扩成 **bank0/bank1/追加表三表分派**(bank1 分支:验 sig=bank1值 + $87/$88∈bank1表范围 + 查表)。
2. **表放不进 91B 安全区**:表驱动循环代码本身 ~85B(base 41 + loop 38 + tail 6),$5FA5-$5FFF 仅剩 ~6B,**放不下表**。CG 场景写 $5C00-$5FA4 → 不能放那。**表须置于运行时可映射的空闲区**:候选=bank0/bank1 句表窗口($A000-$BFFF)里的空隙(钩子运行时 $5115 已映射),或另辟固定 bank。待勘查句表 bank 的空闲字节。
