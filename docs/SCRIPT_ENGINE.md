# 脚本引擎逆向(白盒反编译进行中)

目标:直接从脚本 bytecode **静态推导完整遍历图**(所有场景/对话/分支),取代动态黑盒遍历。
判定实验证明游戏是"脚本流"(状态路径相关,无法状态签名去重),故动态遍历只能穷尽采样;
而 bytecode 是确定的,静态反编译能给出**可证完整**的遍历图。本文记录已确认的引擎结构。

## ★两套独立系统(关键认知)

第一章的流程由**两套互不相同的系统**驱动,遍历图分两半:

| 系统 | 管什么 | 遍历图的哪一半 |
|---|---|---|
| **过场脚本解释器**（`$8012` 主循环）| 场景切换、过场动画、CG、区域设置 | **场景推进 / 「前往」**（切场景命令）|
| **对话菜单系统**（`$87/$88` 句子指针 + `$2C` 光标 + 菜单派发）| 场景内话题菜单、对话显示 | **场景内话题分支** |

实测:海边菜单交互全程 `$8012` 命中 **0 次** → 对话菜单**不经过**脚本解释器。两套要分别逆向。

## 一、过场脚本解释器(bytecode,已基本摸清)

- **脚本指针** `$C6/$C7`(CPU 地址,走 `$A000-$BFFF` 窗口)、**脚本 bank** `$C5`（写 `$5115`）。
- **主循环** `$8012`：`LDA ($C6),Y` 读命令字节 → 高 4 位查类别处理表 `$8079` → `$8099[高4位]`
  决定"低4位是子命令(=0)还是参数(≠0)" → 取 3 字节项(长度+处理程序地址)→ 读参数 →
  `JMP ($00B9)` 跳处理程序 → 处理完 `JMP $8012`,指针已 `+= 命令长度` 前进到下一条。
- **命令表**：类别处理表基址 `$8079`(16×2B)、子偏移 `$80A9`(`00 03 06…2D`,每项 3B)。
- **已确认命令**：

| 命令 | 作用 |
|---|---|
| `$01 <ofs16>` | **调用 CALL**：存返回 `$C6/C7→$C8/C9`,再 `$C6 += ofs`（相对）|
| `$02 <ofs16>` | **跳转 JMP**：`$C6 += ofs`（相对偏移,非绝对地址）|
| `$60-$6F` / `$70` / `$80` / `$B2` | **切场景**：`LDA $BB; STA $0450`（`$BB`=参数=目标场景 bank）= **「前往」的底层** |
| `$B4` / `$B5` | 设句子指针 `$87/$88`（触发对话显示）|

- **切场景处理程序** `$8307`：`LDA $BB; STA $0450; JMP $8012`。这就是黑盒试探认不出、代码里
  一清二楚的"前往"——一条脚本命令把目标场景号写进 `$0450`。

## 二、对话菜单系统(核心在 bank61,已挖到数据层)

菜单光标 `$2C` 的读写全在 **bank61** → 对话菜单系统在 bank61。

- **菜单主循环** `$885A`：`JSR $E057` 读手柄 → `$062F` 按键 `AND #$CF` 分派:
  `$80`(A)→`$88A0` 选中 / `$40`(B)→`$88F5` 返回 / `$08/$04/$02/$01`(上下左右)→`$890C`+ 导航。
- **A 键选中派发** `$88A0`：`LDX $21`(层深)→ `$2C`(光标)`×2 + $0529,X`(本层选项表基址)
  → `LDA $04E8,Y` 取选项的目标指针 → `INC $21`(层深+1,深入)。
- **菜单栈**(层级结构):`$21`=当前层深;每层一套 `$0529/$052E/$0533,X`(选项表基址/指针)、
  `$0542,X`(该层选中项);**`$04E8` = 选项指针表**(每项 2 字节,指向选项定义块)。
- **选项表填充** `$83EA`:从 `$1E/$1F`(指向菜单定义)读选项块链,每块头 2 字节=步进长度
  (`$64/$65`),块地址存入 `$04E8`,`$1E += 长度` 到下一块。

### ★菜单定义数据格式(遍历图的源头,动态解码得)

菜单定义在 **`$A000-$BFFF` 窗口**(`$5115` 映射,菜单态实测 bank=143,≠脚本 bank `$C5`;
**只能动态读或先读 `$5115` 再定位,勿按 `$C5` 静态算**)。实测海边(`$04E8`=`$BF5B/$BFA5/$BFEF`):

- **每个选项块** = `标签句号(2字节小端,Lua系)` + `内容 bytecode`。
  标签验证:`$BF5B`→句94=查看、`$BFA5`→句97=交谈、`$BFEF`→句102=回头看(Lua 94/97/102 = tr 95/98/103)。
- **内容 bytecode** = `前缀字节 + 标签句号` 序列:
  - **`$50 <句号>` = 一个话题子选项**:查看块含 `50[105大海] 50[106山丘那边] 50[107机场]`。
  - 其他前缀 `$4A/$02/$C1/$0A/$1E/$52/$51…` 区分选项类型(话题/对话/子菜单/跳转),**语义待逐个解**。
- **★「前往」= 菜单标签句 100**(在交谈块 `+15:[100=前往]`)。回答了核心疑问:**"前往"不是特殊
  机制,就是菜单里的一个选项**;选它 → 触发切场景(§一的 `$8307` 命令)。

### ★★菜单选项提取器已打通(核心机制验证)

选项在 `$04E8` 表的位置 = **`$0529[$21]`(当前层基址,累积)+ 光标×2**;`$0528` 是累积写入位。
读 `$04E8[base + i×2]` → 选项块地址 → 块头 2 字节 = 标签句号。递归各层 = 遍历图选项树。

**实测海边(逐层提取,句号全对)**:
- 顶层:`94/97/102` = 查看 / 交谈 / 回头看
- 查看 → `105/106` = 大海 / 山丘那边
- 交谈 → `99/96` = 梓 / 艾莉娜

工具雏形 `scratchpad/mnav.py`（动态导航各层读 `$04E8` 标签句）。**"直接从菜单数据提取选项树"已经能做到。**

## ★★★★★纯静态选项解析器达成(`reversing/senmap/staticmenu.py`)

**动态 DOWN 测的病根**:靠"跑游戏按按钮、看光标动不动"**猜**选项类型,选项语义多样时猜错、漏项
(海边查看只探到大海/山丘,漏机场)。**正解 = 解菜单定义数据本身的语法**(数据是确定的,不用猜)。

**菜单定义语法(已破解)**:
- 菜单 = **顶层命令块链**(查看/交谈/回头看/休息…),块地址存 `$04E8` 指针表;块长 = 相邻指针差。
- 每块 = `标签句号(头2字节小端,Lua系)` + `内容 bytecode`。
- 内容里 **`$50 <标签句>` / `$02 <标签句>` = 一个子选项**(`$50`=看景/话题,`$02`=看人/导航);
  句100=前往(切场景)、句102=回头看;中间夹坐标命令(`$01/$05/$C1/$0A`)与对话 bytecode。
- 解析:按块边界切,块内扫 `$50/$02 <标签>`,**去重 + 剔块自引用 + 剔对话句(带「)+ 剔拉丁(职员表越界)**。

**实锤:纯静态 > 动态(处处更完整,一项不漏且不用猜)**:

| 场景 | 静态解析(staticmenu) | 动态 DOWN 测(menutree) | 静态多找出 |
|---|---|---|---|
| 0B 海边 | 查看→大海/艾莉娜/经销店/山丘/机场 | 查看→大海/山丘 | **机场/艾莉娜/经销店** |
| 0E 工作间 | 交谈→艾莉娜/阿源/机器人的事/查米的事/留言的事 | 交谈→艾莉娜/阿源→机器人/查米 | **留言的事**、查看多**到外面去** |
| 14 阿源车 | 交谈→艾莉娜/梓/兴奋雀跃 | 交谈→艾莉娜/梓 | **兴奋雀跃** |

**回答了"都读懂源码了还不能静态遍历出所有句子?"——能。** 选项层面纯静态、完整、确定。
现为"半静态"(动态读一次菜单定义字节,避开 `$5115` bank 映射定位);纯 ROM 定位需另解 bank 映射。

**剩余(选项→逐句)**:`$50 <标签>` 后跟的响应句、及"父母的事/留言的事"这类是否再开子块,
是块内下一层语法(响应句 / 子块指针),解开即可从"选项树"推进到"选项→完整对话"的纯静态逐句图。

## ★★★★★★对话演出引擎全解(选项→对话的完整机制,数据流实证)

用"数据流追踪(动态)+ 反汇编(静态,`reversing/tools/disasm6502.py`)"两手,把"选大海→句182"
从头到尾追通。**结论:选项块内容本身就是脚本,选中即把脚本指针指向它并跑解释器。**

### 1. 真正的对话演出解释器在 `$EDE5`(不是 `$8012`)

- 之前发现"菜单交互 `$8012` 命中 0 次"——因为对话演出走**另一个解释器 `$EDE5`**,脚本指针是
  **`$1E/$1F`**(bank 在 `$20`),不是 `$8012` 那套 `$C6/$C7`。这个 `$1E/$1F` 正是菜单填充选项表
  (`$83EA`)用的指针——**菜单与演出共用 `$1E/$1F`**。
- 主循环 `$EDDE`:`JSR $EDE5`(执行一条命令,进位标志控制)`; BCC 继续 ; BCS 重来`。
- `$EDE5` 一条命令:`LDA ($1E),Y` 读命令字节 → `(c>>5)&6` 选处理表 `$8004,X`(bankBD)→
  `c*4` 索引 → 读参数(标志位 `$66` 决定读几个参数到 `$5E-$61/$64/$65`)→ `$EE88`:`$1E += 命令长度` 前进。

### 2. 句号 → 句子指针 的公式(`$F01D` 解码器,完全破译)

```
$87/88 = 句号 × 3 + 基址表[$5D1E/$5D21, X]      (每句指针项3字节;X=句子库索引)
地址高3位 → bank(基址表 $5D1B,X)→ 写 $5115 ;  低位 → $A000 窗口地址
$F071: LDA ($87),Y → 读该句3字节(长度+文本地址)到 $80/$81/$82
```
- **句子库入口**:`$EF73`(库0:菜单标签)/`$EF96`(库1)/`$EFB9`(库2:剧情对话);句号从 X/Y 传入。
- **句子链式**:每句文本数据末尾带"下一句句号"(`$91C9` 读 `($29),Y→$12/$13`→作下一句号)→
  一句接一句自动播,直到结束标记(`$81==$1F && $87==$FE/$FF`,见 `$F061`)。

### 3. 选项→对话的完整链(大海实证,句号皆 Lua 系)

```
选中"大海"($88A0 选中逻辑, PC $88B5/$88BD, bank61)
  └→ $1E/$1F = $BF60 (大海选项块地址,从 $04E8 取)   ← 关键:脚本指针指向选项块
  └→ 跳过块头,$EDE5 解释器执行大海块内容 bytecode ($BF64 起, PC $EE8E 逐命令前进)
     └→ 块内命令执行到跳转 → bankA9:$B791 (大海演出脚本流)
        └→ 播 句181「梓「哥哥 这海好吧」忠「嗯」…」 → 链式续播
```

**这解释了为什么选项块里找不到 182**:选项块存的是**脚本命令**,对话在被执行的演出脚本流里。
`$8012 命中 0 次`也解释了:演出走 `$EDE5`/`$1E-1F`,与 `$8012`/`$C6-C7` 是两套并行解释器。

### 4. 通往纯静态反编译器(剩余)

要纯静态推导"选项→完整对话",只差**完整解码 `$EDE5` 命令集**(命令字节 → `$8004,X` 处理表 →
参数/跳转/句号语义)。已知命令 `$02`(参数=看人选项句号 96/95/94)、`$01/$05/$C1`(疑坐标/布局);
解全 → 从选项块 bytecode 静态解释 → 跟随跳转到演出脚本流 → 展开全部对话 = 纯静态逐句遍历图。
**这是"完整反编译器"的核心,现已定位到确切的解释器与命令分派表,不再是黑箱。**
