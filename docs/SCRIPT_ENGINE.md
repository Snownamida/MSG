# 脚本引擎逆向 —— 从黑箱到白盒反编译

记录 MSG 脚本/对话演出引擎的完整逆向结论。目标:**直接从 bytecode 静态推导对话/遍历图**,
取代靠跑游戏的黑盒穷尽。核心工具在 `reversing/tools/`(`disasm6502` 反汇编、`mesen` 无头驱动)
与 `reversing/senmap/`(`scriptdis`/`autoscript`/`scandump` 等)。

## 核心结论(TL;DR)

- **游戏是"脚本流"**:状态路径相关(同场景不同进度对话不同),无有限状态可签名去重 →
  动态遍历只能按路径采样、有"漏句"隐患;而 bytecode 确定 → **静态反编译可给全集,天然不漏**。
- **两套并行解释器**:过场脚本 `$8012`(场景切换)与对话演出 `$EDE5`(场景内对话),互不经过。
- **白盒反编译已达成**:选项 → 对话的完整链路逐字节可读,**连对话内容都能纯静态从 bytecode 提取**
  (实证:海边"看"演出句 181-216、工作间"看阿源"句 134-137,与运行时一致)。
- **完美遍历正解 = 静态反编译出句子全集(不漏) × 动态遍历定实际路径/画面(验证)**,两法互补。

## 一、总体架构

### 两套并行解释器

| 系统 | 脚本指针 | 管什么 | 遍历图的哪半 |
|---|---|---|---|
| **过场脚本** `$8012` | `$C6/$C7`(bank `$C5`) | 场景切换、过场动画、CG、区域设置 | 场景推进 / 「前往」 |
| **对话演出** `$EDE5` | `$1E/$1F`(bank `$20`) | 场景内话题菜单、对话演出 | 场景内话题分支 / 对话 |

实测海边菜单交互全程 `$8012` 命中 **0 次** → 对话演出**不经过**过场解释器,两套分别逆向。

### 文本寻址:句号 → 句子指针(`$F01D` 解码器,完全破译)

所有对话/脚本都经统一寻址。`$F01D` 把"句号"翻译成 CPU 地址:

```
t   = 句号 × 3 + (base_lo | base_hi<<8)          每句指针项 3 字节
指针 = $A000 | (t & $1FFF)                        映射到 $A000-$BFFF 窗口
bank = ((t>>13) & 7) + base_bank → 写 $5115
$F071: LDA (指针),Y → 读该句 3 字节(bank/长度 + 文本地址)到 $80/$81/$82
```

- **三句子库**:入口 `$EF73`(库0:菜单标签 + **演出脚本入口表**)/`$EF96`(库1)/`$EFB9`(库2:剧情对话);
  句号从 X/Y 传入。base 三张表 `$5D1B`(bank)/`$5D1E`(lo)/`$5D21`(hi),按库 0/1/2 索引。
- **base 表全局固定**(实测场景 0B 与 0E 相同 `80 80 81 · 8C B5 4C · 00 1C 1D`)→ 句号→地址是全局映射;
  只需每场景动态读一次 `$5D1B-$5D23` 即可静态复现。演出中零写入(不被改)。
- **句子链式**:每句文本数据末尾带"下一句句号"(`$91C9` 读 `($29),Y→$12/$13` 作下一句号)→ 一句接一句
  自动播,直到结束标记(`$81==$1F && $87==$FE/$FF`,见 `$F061`)。

## 二、对话演出引擎 `$EDE5`(第一章对话的核心)

### 解释器与命令分派

- 主循环 `$EDDE`:`JSR $EDE5`(执行一条命令,进位标志控制)`; BCC 继续 ; BCS 重来`。
- 一条命令 `$EDE5`:`LDA ($1E),Y` 读命令字节 `c` → 命令分派 → `$EE88`:`$1E += 命令长度` 前进。
- **命令分派机制**(处理表在 bankBD 的 `$8000` 窗口):
  ```
  类别 = (c>>5)&3 → 处理表基址 = $8004+类别*2  (4 类别基址 $800C/$810C/$8168/$8268)
  表项 = 基址[(c*4)&$FF]:  +0=$13(属性)  +1=$66(流内参数标志)  +2/+3=处理程序地址→$1A/$1B
  分派: JMP ($1A)          每命令跳独立处理程序(可在任意 bank)
  ```

### 命令集

| 命令 | 作用 |
|---|---|
| `$4E <句号lo hi>` | **显示对话**(处理程序取句子显示;实证 `4E B5 00`→句181 大海对话) |
| `C1 <lo hi>` | **跳转到演出脚本**(见下,把库0句号定位的地址设为新脚本指针) |
| 库切换(处理程序 `$EECE`) | 从脚本流 `$A000-$A00B`(12 字节库描述符=3库×bank/lo/hi)重载 `$5D1B/$5D1E/$5D21` 三基址表 |
| `$02 <句号>` | 看人选项 · `$01/$05/$C1` 布局/坐标 |

**命令长度**:多数 = `1 + ($66&$08?1:0) + ($66&$80?1:0)`;部分处理程序自读 `$1E`(如 `$4E` 读 2 字节句号),
动态 trace 实测补全:`$00=2 $01=2 $02=3 $03=2 $05=2 $1D=1 $27=1 $3C=3 $40=2 $41=1 $49=2 $4E=3`。

### 演出脚本流格式

演出区是"剧情句子按号线性存储"的数据,每条对话 6 字节:`$40 00`(开框) · `$4E <lo hi>`(显示句号) · `$41`(收尾)。
段内句号连续、段间大跳(=不同剧情)。海边"看/谈"是一条连续流(句 181-216:看海→机场→爸妈→查米→阿源→车)。

### C1 跳转:选项块 → 演出脚本入口(全静态链已验证)

`C1` 处理(`$F38F`):`LDX $64; LDY $65`(读的 2 字节)→ `JSR $EF73`(**当作库0"句号"取其文本地址**)
→ `STX $1E; STY $1F; STA $20`。**惊人的统一:演出脚本本身就存在句子库里当"句子",C1 用句号经 `$F01D`
定位演出脚本入口** —— 脚本与对话共用一套寻址,库0 就是"演出脚本入口表"。

```
海边"看" C1 库0句号 94 →$F01D(海边base)→ 句子指针 bank80:$A1A6
                      →读 3 字节 = [29 91 17]
                      →$EF73 解读: bank=$29|$80=A9, addr=$91|(($17+$A0)<<8)=$B791
                      → 演出脚本入口 bankA9:$B791  ✓ 与运行时完全一致
```

## 三、对话菜单系统(bank61)

菜单光标 `$2C` 的读写全在 bank61。

- **主循环** `$885A`:`JSR $E057` 读手柄 → `$062F & $CF` 分派:`$80`(A)→`$88A0` 选中 /
  `$40`(B)→`$88F5` 返回 / 方向→`$890C` 导航。
- **A 键选中** `$88A0`:`LDX $21`(层深)→ `$2C×2 + $0529,X` → `LDA $04E8,Y` 取选项块指针 → `INC $21` 深入。
- **菜单栈**:`$21`=层深;每层 `$0529/$052E/$0533,X`(选项表基址)、`$0542,X`(选中项);
  **`$04E8` = 选项指针表**(每项 2 字节 → 选项定义块)。
- **选项表填充** `$83EA`:从 `$1E/$1F`(菜单定义)读选项块链,块地址存 `$04E8`,`$1E += 块长`。

### 菜单定义格式:选项块即脚本

菜单定义在 `$A000-$BFFF` 窗口(`$5115` 映射,菜单态实测 bank≈143,**动态读或先读 `$5115` 定位**)。

- **每个选项块** = `标签句号(头 2 字节小端,Lua 系)` + `内容 bytecode`。
  实测海边:`$BF5B`→句94=查看、`$BFA5`→句97=交谈、`$BFEF`→句102=回头看。
- **内容 bytecode = `$EDE5` 解释器的脚本**:`$50 <标签句>` / `$02 <标签句>` = 子选项(`$50`看景/话题、
  `$02`看人/导航);句 100=前往(切场景)、102=回头看;夹坐标命令(`$01/$05/$C1/$0A`)。
- **选中 = 把脚本指针 `$1E/$1F` 指向选项块并跑 `$EDE5`**(`$88A0` PC `$88B5/$88BD` 设 `$1E`);
  块内 `C1` 命令再跳到演出脚本流。这解释了为什么"选项块里找不到对话句"——对话在被执行的演出脚本里。
- **「前往」不是特殊机制**,就是菜单里标签句 100 的选项,选它触发切场景。

## 四、过场脚本解释器 `$8012`(场景切换)

- 脚本指针 `$C6/$C7`(`$A000-$BFFF` 窗口)、bank `$C5`(写 `$5115`)。
- 主循环 `$8012`:`LDA ($C6),Y` 读命令 → 高 4 位查类别表 `$8079` → `$8099[高4位]` 定"低4位是子命令/参数"
  → 取 3 字节项(长度+处理地址)→ `JMP ($00B9)` → 处理完 `JMP $8012`,指针已前进。

| 命令 | 作用 |
|---|---|
| `$01 <ofs16>` | CALL:存返回 `$C6C7→$C8C9`,`$C6 += ofs`(相对) |
| `$02 <ofs16>` | JMP:`$C6 += ofs`(相对) |
| `$60-$6F`/`$70`/`$80`/`$B2` | 切场景:`LDA $BB; STA $0450`(处理程序 `$8307`)= 「前往」底层 |
| `$B4`/`$B5` | 设句子指针 `$87/$88`(触发对话) |

## 六、关键结论与方法论

### 脚本流本质:静态=全集 ⊇ 动态=路径子集

静态 `autoscript` 提出海边剧情段 **句181-216 全集**;动态遍历(`ch1_sentence_scene`)在同段只碰到
**181,182,184,185,186,188,189**(跳着的)。**动态是静态的路径子集** —— 演出区是线性存储的数据全集,
动态只走到运行时路径碰到的那些。这就是"暴力遍历会不会漏"的终极解答:静态从数据区给全集、天然不漏。

### 全章转储方法(两场景验证通用)

**动态抓选项触发的演出脚本地址 `$1E/$20`(各场景不同)× `scandump`/`autoscript` 静态展开**:
- 海边"看"→ `A9:$B791` → 句 181-216;工作间"看阿源"→ `A5:$BF65` → 句 134-137。
- 第一章主线剧情是**一条连续演出流(句181-216)**,各场景各选项从不同偏移进入(0B→181、0E→201、11→190、17→191)。

全章逐句转储 = 对各场景各选项重复此法的枚举工程,机制与工具链均已就绪。

### 现状

引擎架构、句号寻址、两套解释器、命令分派与命令集、菜单定义格式、演出脚本流格式、C1 跳转、全静态对话
提取——**全部打通并有工具**。第一章"完美遍历"由动态方案(`seqrun × traverse × audit`)达成并 PASS;

纯静态化、条件分支命令语义。
