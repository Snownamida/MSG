# Metal Slader Glory — 文本引擎逆向笔记

面向中文化的静态逆向成果。地址约定：`PRG` = PRG ROM 内偏移（= .nes 文件偏移 − 0x10）；`CPU` = 6502 地址。
标注**确认**（从指令直接读出/与 `msgtool.py` 交叉验证）或**推测**。

## 卡带 / MMC5 配置（确认）

- Mapper 5 (MMC5)，PRG 512KB + CHR 512KB，带电池 SRAM，水平镜像。
- 模式寄存器（reset 于 `$E94E` 设置）：`$5100=3`（PRG 8KB×4）、`$5101=1`（**CHR 4KB banking**）、`$5104=2`（ExRAM 作普通工作 RAM）、`$5105` nametable 映射。
- PRG 窗口：`$E000-$FFFF` = `$5117` = **固定 bank 63**；`$A000-$BFFF` = `$5115` = 文本解引用时按需切换的数据窗口；`$8000`/`$C000` 也动态切。
- 向量：`NMI=$E9C3  RESET=$E94E  IRQ=$F978`（均在 bank 63）。
- **可执行代码只在 bank 59–63（PRG `0x76000–0x7FFFF`，顶部 40KB）**，其余 472KB 全是数据（CDL code 标记佐证）。

## tile 映射（确认：恒等）

**PPU 编码字节 b → 字库 tile 号 = b，无偏移、无查表。** 逐环验证：ROM 的 PPU 串字节 → 行缓冲 `$0469` → 字符对缓冲 `$04A9,X`（`$F176: STA $04A9,X`，前无算术）→ VRAM 队列 `$0300`（直拷）→ `$2007`（`$F8F6: LDA $0300,X; STA $2007`）。
字库 = 一个 CHR 4KB bank（256 tile），tile 排列 = `msgtool.py` 的 `CHARSET` 顺序（渲染验证：tile 0x20='0'、0x2A='A'、0x50='あ'、0x0E='゛'、0xFE=空白）。

## 屏上字符几何（确认）

每个对话字符 = **上下两个 8×8 tile 竖排**（cell 8×16，字形在下格）：
- 渲染为每字生成一对 `[mark, char]` 存入 `$04A8`（偶=上格 tile，奇=下格 tile）。
- `char` = 字形 tile（恒等）；`mark` 默认 `0xFE`（空白），仅当 PPU 串遇 `0x0E`/`0x0F` 前缀时置为 `゛`/`゜`。
- **浊音/半浊音 = 把 ゛/゜ tile 摆到假名上方那格**（硬件叠放，非合成字形）。

## 例程地图（确认）

| 例程 | CPU / PRG | 作用 |
|---|---|---|
| 文本渲染核心 | `$F0B7` / `0x7F0B7`（备用入口 `$F0C0`） | 句块串 → 块 → PPU 串 → `$04A8` 对缓冲 |
| level-1 指针解引 | `$EF96` / `0x7EF96` | 从 `$29/$2A` 取块串指针 |
| block→文本指针 | `$EFB9` / `0x7EFB9` | 内部调 `$F01D` |
| ×3+基址+切bank | `$F01D` / `0x7F01D` | text_pointer；用 ExRAM `$5D1B/$5D1E/$5D21` 基址表，切 `$5115` |
| **string_pointer 解包** | `$EFC3` / `0x7EFC3` | **与 msgtool `string_pointer` 逐位一致**：`bank=b1&0x3F→$5115`、`ptrHi=(b3&0x1F)+0xA0`、`len=((b1>>6)<<3)+(b3>>5)` |
| 拷 PPU 串到行缓冲 | `$EFEB` / `0x7EFEB` | ROM `($81),Y` → `$0469,X`；首字节 0 的特例同 msgtool |
| 字符发射循环 | `$F135–$F185` | `$0469` PPU 字节 → `$04A8` `[mark,char]` 对；`$27` 锁存浊音前缀 |
| 矩形缓冲→VRAM队列 | `$E270` / `0x7E270` | `$04A8` → `$0300` 队列（参数块 `$04A3..`：宽、行数=2、PPU 地址、flags） |
| VRAM 队列刷新 | `$F8AA` / `0x7F8AA` | vblank 内把 `$0300` 队列写 `$2007`，预算 `$0449=#$4C`（76B/帧）；包格式 `[count][addrHi][addrLo][data…]` |
| 行排版/分页 | `$A4D7`（bank61 `0x7A4D7`） | 走块串、处理控制码、按 `#$1E`(30 tile) 宽度断行、建行指针表 `$04E8`（≤8 段） |
| NMI | `$E9C3` | OAM DMA；调色板上传；MMC5 行 IRQ 设置（`$5203/$5204`）；载入 CHR bank（`$EB5B`: `$0450→$512B/$5123`, `$0453→$5127`） |
| IRQ（光栅分屏） | `$F978` | 经 `($0633)` 分派；分屏应用器 `$FF26` 从 `$045E,Y→$512B/$5123` 逐区切 CHR/滚动 |

### 渲染核心伪代码
```
render_line($F0B7):                     # $29/$2A=行块串指针, A=起始列*2
  $27 = 0xFE
  ($18,$19) = resolve_ptr($29/$2A)      # $EF96
  loop over block-string ($18),Y:       # $F0D7
     b = read()
     if b==0x00: finalize()
     elif b>=0x80: block=(b<<8)|next    # 双字节块(MTE)
     else:         block=b
     expand_block(block) -> $0469[]     # $EFB9: block→text→string→拷PPU串
     for each PPU byte c in $0469:       # $F135..$F185
        if c in {0x0E,0x0F}: $27=c; continue   # 锁存浊音到上格
        $04A8[k]   = $27 ; $27 = 0xFE          # 上格 = 标记/空白
        $04A9[k+1] = c                         # 下格 = 字形 tile（恒等）
        k += 2
  # $E270 把 $04A8 作 2 行 blit 进 $0300 队列 → NMI 刷 $2007
```

## CHR bank（确认）

- CHR 4KB 模式：512KB = **128 个 4KB bank，每 bank 256 tile**。
- 文字在背景层，字库 bank = 写 `$512B` 的值；分屏表 `$045B-$0468`（NMI 从 `$0450-$045A` 拷来），IRQ 应用器 `$FF34` 写 `$512B`。复位默认字库 = bank 0。
- ⚠ **修正（PoC 实测）**：先前"对话框区独占 256-tile 页"的说法**不成立**。开场机甲幕 / 海边幕的 CG 与对话字库**共享同一个 bank**，且该 bank 几乎全满（纯空白 tile 仅 1–3 个）——tile `0xB0–0xC5` 在 bank 0/11/48 里都是 CG 图形块（`ink=64` 满格）。**这些场景没有空位无损植入新字模**，详见下方 PoC 坑 3。

## 控制码（推测）

块串层控制码（05/07/0B/0C/13/14/16/18 双字节；0F/10/12/17 三字节）**不在**字形渲染例程分派——核心 `$F0D7` 只认 `0x00`/`≥0x80`/其余。由**上游排版层（bank 61/62，入口 `$A4D7`）经跳转表处理**，渲染前已把句子切成行、建好 `$04E8` 行指针表。故推断为**行/页级**语义：换行、翻页等待、光标定位、角色名、立绘、颜色等（精确含义待追 bank 61/62 跳转表）。宽度上限 `#$1E`=30 tile/行、≤8 行段。

## 中文化改造评估

**有利（确认）**：tile 映射恒等（无需改字节→tile 逻辑）；字库本就可 bank 切换、对话框区独占 256-tile 页；引擎已是"每字一格、可带上格标记"结构；数据侧回写数学已由 msgtool 备齐。

**瓶颈**：任一光栅区一时刻只能寻址 1 个 4KB bank = 256 tile；CHR 是 ROM 不能运行时拼。中文 1500–3000 字远超 256。

**路线（按性价比）**：
1. **MMC5 扩展属性模式（`$5104=1`，ExGrafix）** — 最正统。ExRAM 为每 tile 提供 CHR bank 高位，逐 tile 选 bank，整 512KB CHR 当大字库（《恶魔城传说》日版、《Just Breed》同款）。改动：(a) `$5104` 切扩展属性；(b) **把 ExRAM 工作变量（`$5C00-$5FFF`，含 `$5D1B/1E/21` 文本基址表、`$5DC0-$5EF0` 对象数组等上百处）整体迁到 `$6000-$7FFF` 电池 WRAM**；(c) 每写 tile 号时同步写对应 ExRAM 属性（bank）。难度：中偏高，主要成本是 ExRAM 搬迁。
2. **每行独立字库 bank**（复用现有分屏 IRQ，不动 ExRAM）：对话框区细分光栅、逐行切 `$512B`；需打包期逐行装箱、多插 IRQ、重排分屏表。难度：中。
3. **字号放大（正交）**：8×8 太小；8×16 利用现有上下双格几乎免费（把中文上格也填字模，宽度不变）；16×16 最清晰但要改排版层 `$A4D7`。

**结论**：数据侧可控（msgtool 已铺路）；主成本在 CHR 寻址扩展，走扩展属性模式是最干净通路，ExRAM 搬迁是一次性重构。渲染核心 `$F0B7` 逻辑清晰、恒等映射、改动面小且集中。属"有明确技术路径、需一次中等规模引擎补丁"，非重写文本系统。

## 概念验证 (PoC) 结论 — 2026-07

把句 62（开场艾莉娜台词）patch 成中文 8×8 字模的实验，在 Mesen 里逐帧核对，确认与暴露：

**确认（实证）**
- **tile 恒等映射**被 Tilemap Viewer 实证：屏上 PPU tile index `$B7` 直接对应 CHR 内 tile `$B7`（bank 内偏移 `0xB70`）。
- **每个场景各自加载字库 bank**（不是全程共用 bank 0）：标题幕 = bank 0、海边幕 = bank 11、**开场机甲幕 = bank 48**。同一码位要在用到它的每个场景的字库 bank 里都写一份字模。
- **字库位平面格式**：plane0 全 `0xFF`；plane1 里 **笔画 = 位 0（色号 1，显示为紫）、背景 = 位 1（色号 3，显示为黑）**。造字模若写反（笔画落位 1）→ 颜色反相成"紫底黑字"。正确写法：plane1 先全 `0xFF`，再在笔画像素处 **清零** 对应位。
- 8×8 字模注入正确 bank + 就地改文本 → 模拟器正确显示中文（黑底紫字，与旁边日文一致）。**管线打通。**

**三个坑（决定下一步）**
1. **8×8 太糊**：笔画多的字（觉/意/源）在 8×8 下糊成一团、难辨认。必须上 **16×16** → 需引擎改造（见上文路线 1/3）。
2. **字典字符串重叠**：block `007A`/`0066`/`818F` 等的字符串指向 ROM 里**重叠的同一段字节**（如 `0x056021-22`，极致 MTE 压缩）。就地覆写一个 block 会踩坏共享字节的邻居，导致译文错位（实验中「从/的」错显成「得/觉」）。**结论：回写工具必须整体重建字符串数据区，绝不能就地覆写。**（已由 `reinsert.py` 解决：新数据全部重建到空闲区。）
3. **CG 与字库是不同的 CHR bank（分屏）；最初"机甲变中文"是写错了 bank**（已由实机 + 渲染定论，推翻本坑早期"共享全满"的推断）：
   - **对话框字库 = bank 0**；**机甲 CG = bank 48**；两者靠光栅分屏各用各的 bank。铁证：tile `$83`（片假名 エ）在 **bank 0 是清晰字形、在 bank 48 是 CG 噪声**——对话框显示出 エ，故对话框读 bank 0。
   - ⚠ 工具坑：**Mesen Tilemap Viewer 的 CHR 地址不追 MMC5 bank**（报 `$0830` 误导以为 bank 0）；**Tile Viewer 的地址才含 bank**（`$30830` = bank 48）。判 bank 用 Tile Viewer。
   - 最初 PoC 往 bank 0/11/48 都写、bank 48 是 CG → 机甲变中文。**只往 bank 0 写 → 对话框中文、机甲完好、零副作用**（已实机验证）。

## 里程碑 2：单句正确回写（实机验证）— 2026-07

`reinsert.py` 把句 62 正确回写为中文并在 Mesen 跑通：重建到空闲区（避开字典重叠）+ text 指针反解 + 块串就地重建 + 字模只写 bank 0。**对话框清晰中文、机甲 CG 毫发无损、其他句零副作用**。单场景 demo **无需扩 CHR / 切 bank / 改引擎**。

**但两个上限仍在，全量翻译绕不开引擎改造（B）：**
- **256 tile/bank**：bank 0 已近满，demo 是覆盖了原版低频汉字位（`0xB0–0xC5`="以井宇…"）塞中文——别处日文用到这些字会跟着变（demo 可接受，全量不可）；1500+ 唯一字装不下一个 bank。
- **8×8 太糊**：清晰需 16×16。
两者都要靠"扩 CHR（已验证 1MB 可行）+ 文本段切专属新 bank"解决。

## 里程碑 3：B 命门打通——文本段切到扩 CHR 专属 bank（实机验证）— 2026-07

**对话框是分屏的 region-1**（agent"整屏 bank48"被实机推翻）：region-0（上，机甲）bank = `$0450` = 48；region-1（下，对话框）bank = `$0451`（复位默认 0）。NMI 每帧把 `$0450/$0451` → 影子 `$045E/$045F`，IRQ 按区把 `$045E,Y` → `$512B`。

**打通方法（单处等长改）**：`.nes 0x7EB8F` 的 `AD 51 04`（LDA $0451）→ `A9 80 EA`（LDA #128; NOP），使 region-1 影子 `$045F` = 128。于是对话框读扩 CHR 的 **bank 128**，机甲（region-0 = 48）与 bank0 均不受影响。

**实机验证**（`build_b_demo.py` → `MSG-b-demo.nes`）：中文只放 bank128、bank0 原封不动 → 对话框显示正确中文、机甲 CG 完好。**证明文本段可切到扩 CHR 的专属新 bank。**

→ **至此全部技术难关验证通过**（导出／回写／字库／bank 机制／扩 CHR／文本段专属 bank），剩余为工程量：全量翻译、全量回写、多 bank 字库管理（>256 字）、16×16 渲染（改排版层 `$A4D7`）。

## 16×16 改造方案（agent 静态逆向）— 2026-07

**结论：只改一个例程（字符发射循环 `$F135–$F185`），原地覆写，其余零改动。**

为什么这么省：引擎当前已是"2 tile 高"渲染（8×16 = 上格标记 + 下格字形，本质 2 行 × N 列 blit，`$04A4`=行数恒 2）；VRAM 队列 `$E270` 是**宽度无关的垂直条带 blitter**（dialog 用 `flags=$01` 转置，按 2 字节/条带铺 `width` 列，+32 垂直递增）；宽度 `$04A3` 由 `$A6D2: LDA $14; LSR` 自动推导。所以 16×16 只是"每字吐 4 tile + 每字占 2 列"，`$E270`/参数块/NMI/断行阈值全不动。

**emit 新逻辑**（每字 4 字节 `[TL,BL,TR,BR]`，游标 `$14` +4，列 `$1A` +2；删掉浊音处理腾空间；~62 字节 ≤ 原 80）：
```
读 c ($0469,Y) → base=c<<2 (ASL ASL) → 写 base,base+1,base+2,base+3 到 $04A8,X
测量路径(MEAS)与发射路径都 +2 列/字 → #$1E=30 列阈值不变 = 15 字/行
```
- **字库排布**：每字 4 tile 连排 `4c=TL, 4c+1=BL, 4c+2=TR, 4c+3=BR`；`c<64`（**每 4KB bank 只装 64 字**，密度 256→64）。PPU 码位从"tile 号"变成"字号 c"。
- **代码空间**：emit 原地放下（62≤80，固定 bank 63），无需跳板。
- emit 起点 `.nes 0x7F145`（PRG 0x7F135）。

**风险点（agent 标注，待实机确认）**：
1. `$F0B7` 有第二个 caller `$CC1B`（bank 62，`.nes 0x7CC2B`）也按 2 字节/格填 `$04A8` 并比较 `$0557,X`——改 emit 为 4 字节/字会影响它。需判定它是对话文本（一并 16×16）还是独立 8×16 子系统（需模式标志门控）。**唯一可能扩大工作量的因素。**
2. 打字机逐字显示慢约 2×（VRAM 数据翻倍，观感非 bug）。
3. 行距：行高仍 2 tile，16×16 行可能纵向贴合，观感待核对。

### 全量 B 的技术准备（agent 静态定位，确认级）
- **背景字库 bank 的硬件写入咽喉**：NMI `.nes 0x7EB6B` `STA $512B`（帧起始默认，取自 `$0450`）；IRQ `.nes 0x7FF47` `STA $512B`（分区，取自 `$045E,Y`）。
- **bank 号来源**：场景画面数据流命令 `70 <bank>` → 处理程序 `$8307`（PRG `0x76307`）写 `$0450`；标题 `70 00`/海边 `70 0B`/机甲 `70 30`。
- **改法（方案 A，待实机验证）**：在文本段扫描线分屏处追加 `LDA #<新bank>; STA $512B/$5123`，把文本段切到专属中文字库 bank；帧顶 NMI 用 `$0450` 复位，CG 段不受影响。
- ⚠ agent 曾判"机甲幕整屏单一 bank 48、无 CHR 分屏"——**被实机推翻**（对话框实为 bank 0），说明对话框区存在 agent 静态未捕获的 bank 切换；全量改造前需在此处补动态确认。
