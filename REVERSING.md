# Metal Slader Glory — 文本引擎逆向笔记

面向中文化的静态逆向成果。地址约定：`PRG` = PRG ROM 内偏移（= .nes 文件偏移 − 0x10）；`CPU` = 6502 地址。
标注**确认**（从指令直接读出/与 `msgtool.py` 交叉验证）或**推测**。

## 卡带 / MMC5 配置（确认）

- Mapper 5 (MMC5)，PRG 512KB + CHR 512KB，带电池 SRAM，水平镜像。
- 模式寄存器（reset 于 `$E94E` 设置）：`$5100=3`（PRG 8KB×4）、`$5101=1`（**CHR 4KB banking**）、`$5104=2`（ExRAM 作普通工作 RAM）、`$5105` nametable 映射。
- PRG 窗口：`$E000-$FFFF` = `$5117` = **固定 bank 63**；`$A000-$BFFF` = `$5115` = 文本解引用时按需切换的数据窗口；`$8000`/`$C000` 也动态切。
- 向量：`NMI=$E9C3  RESET=$E94E  IRQ=$F978`（均在 bank 63）。
- **可执行代码只在 bank 59–63（PRG `0x76000–0x7FFFF`，顶部 40KB）**，其余 472KB 全是数据（CDL code 标记佐证）。

## tile 映射（确认：恒等）

**PPU 编码字节 b → 字库 tile 号 = b，无偏移、无查表。** 逐环验证：ROM 的 PPU 串字节 → 行缓冲 `$0469` → 字符对缓冲 `$04A9,X`（`$F176: STA $04A9,X`，前无算术）→ VRAM 队列 `$0300`（直拷）→ `$2007`（`$F8F6: LDA $0300,X; STA $2007`）。
字库 = 一个 CHR 4KB bank（256 tile），tile 排列 = `msgtool.py` 的 `CHARSET` 顺序（渲染验证：tile 0x20='0'、0x2A='A'、0x50='あ'、0x0E='゛'、0xFE=空白）。

## 屏上字符几何（确认）

每个对话字符 = **上下两个 8×8 tile 竖排**（cell 8×16，字形在下格）：
- 渲染为每字生成一对 `[mark, char]` 存入 `$04A8`（偶=上格 tile，奇=下格 tile）。
- `char` = 字形 tile（恒等）；`mark` 默认 `0xFE`（空白），仅当 PPU 串遇 `0x0E`/`0x0F` 前缀时置为 `゛`/`゜`。
- **浊音/半浊音 = 把 ゛/゜ tile 摆到假名上方那格**（硬件叠放，非合成字形）。

## 例程地图（确认）

| 例程 | CPU / PRG | 作用 |
|---|---|---|
| 文本渲染核心 | `$F0B7` / `0x7F0B7`（备用入口 `$F0C0`） | 句块串 → 块 → PPU 串 → `$04A8` 对缓冲 |
| level-1 指针解引 | `$EF96` / `0x7EF96` | 从 `$29/$2A` 取块串指针 |
| block→文本指针 | `$EFB9` / `0x7EFB9` | 内部调 `$F01D` |
| ×3+基址+切bank | `$F01D` / `0x7F01D` | text_pointer；用 ExRAM `$5D1B/$5D1E/$5D21` 基址表，切 `$5115` |
| **string_pointer 解包** | `$EFC3` / `0x7EFC3` | **与 msgtool `string_pointer` 逐位一致**：`bank=b1&0x3F→$5115`、`ptrHi=(b3&0x1F)+0xA0`、`len=((b1>>6)<<3)+(b3>>5)` |
| 拷 PPU 串到行缓冲 | `$EFEB` / `0x7EFEB` | ROM `($81),Y` → `$0469,X`；首字节 0 的特例同 msgtool |
| 字符发射循环 | `$F135–$F185` | `$0469` PPU 字节 → `$04A8` `[mark,char]` 对；`$27` 锁存浊音前缀 |
| 矩形缓冲→VRAM队列 | `$E270` / `0x7E270` | `$04A8` → `$0300` 队列（参数块 `$04A3..`：宽、行数=2、PPU 地址、flags） |
| VRAM 队列刷新 | `$F8AA` / `0x7F8AA` | vblank 内把 `$0300` 队列写 `$2007`，预算 `$0449=#$4C`（76B/帧）；包格式 `[count][addrHi][addrLo][data…]` |
| 行排版/分页 | `$A4D7`（bank61 `0x7A4D7`） | 走块串、处理控制码、按 `#$1E`(30 tile) 宽度断行、建行指针表 `$04E8`（≤8 段） |
| NMI | `$E9C3` | OAM DMA；调色板上传；MMC5 行 IRQ 设置（`$5203/$5204`）；载入 CHR bank（`$EB5B`: `$0450→$512B/$5123`, `$0453→$5127`） |
| IRQ（光栅分屏） | `$F978` | 经 `($0633)` 分派；分屏应用器 `$FF26` 从 `$045E,Y→$512B/$5123` 逐区切 CHR/滚动 |

### 渲染核心伪代码
```
render_line($F0B7):                     # $29/$2A=行块串指针, A=起始列*2
  $27 = 0xFE
  ($18,$19) = resolve_ptr($29/$2A)      # $EF96
  loop over block-string ($18),Y:       # $F0D7
     b = read()
     if b==0x00: finalize()
     elif b>=0x80: block=(b<<8)|next    # 双字节块(MTE)
     else:         block=b
     expand_block(block) -> $0469[]     # $EFB9: block→text→string→拷PPU串
     for each PPU byte c in $0469:       # $F135..$F185
        if c in {0x0E,0x0F}: $27=c; continue   # 锁存浊音到上格
        $04A8[k]   = $27 ; $27 = 0xFE          # 上格 = 标记/空白
        $04A9[k+1] = c                         # 下格 = 字形 tile（恒等）
        k += 2
  # $E270 把 $04A8 作 2 行 blit 进 $0300 队列 → NMI 刷 $2007
```

## CHR bank（确认）

- CHR 4KB 模式：512KB = **128 个 4KB bank，每 bank 256 tile**。
- 文字在背景层，字库 bank = 写 `$512B` 的值；对话框区经光栅分屏拥有**独占 256-tile 页**（不与场景图共享）。分屏表 `$045B-$0468`（NMI 从 `$0450-$045A` 拷来），IRQ 应用器 `$FF34` 写 `$512B`。复位默认字库 = bank 0。

## 控制码（推测）

块串层控制码（05/07/0B/0C/13/14/16/18 双字节；0F/10/12/17 三字节）**不在**字形渲染例程分派——核心 `$F0D7` 只认 `0x00`/`≥0x80`/其余。由**上游排版层（bank 61/62，入口 `$A4D7`）经跳转表处理**，渲染前已把句子切成行、建好 `$04E8` 行指针表。故推断为**行/页级**语义：换行、翻页等待、光标定位、角色名、立绘、颜色等（精确含义待追 bank 61/62 跳转表）。宽度上限 `#$1E`=30 tile/行、≤8 行段。

## 中文化改造评估

**有利（确认）**：tile 映射恒等（无需改字节→tile 逻辑）；字库本就可 bank 切换、对话框区独占 256-tile 页；引擎已是"每字一格、可带上格标记"结构；数据侧回写数学已由 msgtool 备齐。

**瓶颈**：任一光栅区一时刻只能寻址 1 个 4KB bank = 256 tile；CHR 是 ROM 不能运行时拼。中文 1500–3000 字远超 256。

**路线（按性价比）**：
1. **MMC5 扩展属性模式（`$5104=1`，ExGrafix）** — 最正统。ExRAM 为每 tile 提供 CHR bank 高位，逐 tile 选 bank，整 512KB CHR 当大字库（《恶魔城传说》日版、《Just Breed》同款）。改动：(a) `$5104` 切扩展属性；(b) **把 ExRAM 工作变量（`$5C00-$5FFF`，含 `$5D1B/1E/21` 文本基址表、`$5DC0-$5EF0` 对象数组等上百处）整体迁到 `$6000-$7FFF` 电池 WRAM**；(c) 每写 tile 号时同步写对应 ExRAM 属性（bank）。难度：中偏高，主要成本是 ExRAM 搬迁。
2. **每行独立字库 bank**（复用现有分屏 IRQ，不动 ExRAM）：对话框区细分光栅、逐行切 `$512B`；需打包期逐行装箱、多插 IRQ、重排分屏表。难度：中。
3. **字号放大（正交）**：8×8 太小；8×16 利用现有上下双格几乎免费（把中文上格也填字模，宽度不变）；16×16 最清晰但要改排版层 `$A4D7`。

**结论**：数据侧可控（msgtool 已铺路）；主成本在 CHR 寻址扩展，走扩展属性模式是最干净通路，ExRAM 搬迁是一次性重构。渲染核心 `$F0B7` 逻辑清晰、恒等映射、改动面小且集中。属"有明确技术路径、需一次中等规模引擎补丁"，非重写文本系统。
